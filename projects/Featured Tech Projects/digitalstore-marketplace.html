Problem Statement

Existing digital marketplaces like Gumroad and Payhip present several challenges for Indian micro-sellers:

Limited Payment Support

Global platforms rely on PayPal or Stripe, which are not convenient for Indian buyers who prefer UPI, Net Banking, and Razorpay.

Complex Onboarding

Small creators face difficult verification and tax setup processes, making it hard to start selling quickly.

No Localization

Platforms lack support for regional languages, INR pricing, and GST compliance features needed for Indian markets.

High Fees & Delays

Dependence on international accounts leads to high transaction fees, payment delays, and reduced profits for sellers.

Proposed Solution

DigitalStore Marketplace was designed specifically to address the challenges faced by Indian digital creators and buyers with the following key solutions:

Built for Indian Creators

Tailored specifically for small and micro-sellers in India with simplified onboarding and localized features.

Seamless Payment Integration

Supports INR transactions with Razorpay integration, offering UPI, Net Banking, Cards, and Wallet options.

Simplified Seller Onboarding

Quick registration using Email OTP verification instead of complex tax or banking setups.

AI Recommendation System

Hybrid recommendation engine combining content-based and collaborative filtering for personalized suggestions.

Database Schema

The system uses a relational database with the following key tables:

Table Name Purpose Key Fields
users Stores all user accounts user_id, email, password, role
products Digital product information product_id, seller_id, product_name, price
orders Purchase transactions order_id, buyer_id, total_amount, status
order_items Individual products in orders order_item_id, order_id, product_id, quantity
payouts Seller payment records payout_id, seller_id, amount, status
reviews Product ratings and feedback review_id, product_id, buyer_id, rating

AI Recommendation System

The platform includes a hybrid recommendation engine built with Python:

  • Content-based filtering: Uses product attributes (category, tags)
  • Collaborative filtering: Analyzes buyer behavior and purchases
  • Libraries used: Pandas, NumPy, Scikit-learn
  • Integration: Python scripts called from PHP backend
Security Features
  • Password hashing with password_hash()
  • Email OTP verification
  • Session management with secure cookies
  • Input validation and sanitization
  • SQL injection prevention
